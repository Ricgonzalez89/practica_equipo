Este archivo se creo en una rama adicional y luego fue fusionada con la rama Main.

  • git show

    Sirve para mostrar distintos objetos. Por defecto, muestra los cambios históricos realizados sobre los archivos.

  • Datos manipulables con Git

    En Git, lo que se puede modificar es texto plano. Se pueden guardar archivos binarios (como imágenes), pero la mayor precisión en el manejo se alcanza con texto plano.

  • Sistema de Archivos: Linux vs. Windows

    En Windows, las rutas son del tipo: C:/Users/PC/...

    En Linux, son del tipo: /c/Users/PC/...

    En ambos casos, el primer directorio de la ruta se conoce como "raíz": "/" en el caso de Linux y "C:/" en el caso de Windows.

    Al abrir Git Bash, el caracter "~" representan el "hogar (home)" de los documentos, y es la ruta predeterminada donde abre la terminal de Git. Al moverse con "cd" hacia alguna ruta por debajo del home, este caracter siempre representará la primera porción de las rutas subsiguientes (y puede usarse para volver al home, con "cd ~"), a menos que nos movamos a alguna ruta distinta al mismo nivel o por encima del home.

  • ls

    Imprime los directorios y archivos de la ruta actual. Si se especifica el argumento -a, se imprimen todos los directorios incluidos los ocultos; el argumento -l, imprime la salida como una lista. Ambos argumentos pueden darse a la vez: ls -al.

  • mkdir x

    Crea una carpeta con nombre "x" en la ruta actual.

  • Sobre las rutas y su escritura...

    Windows no distingue entre mayúsculas y minúsculas, a diferencia de Linux y MacOS. Al presionar la tecla "Tab", las rutas pueden ser autocompletadas siempre que coincidan con una ruta dentro del directorio en done uno se encuentre.

  • cat x

    Imprime en consola el contenido de un archivo "x".

  • touch x

    Crea un archivo "x". Debe incluirse la extensión del archivo, ya sea .txt, .doc, .css, .html, etc. Solo sirve en entornos Unix, como Git Bash.

  • git x --help

    Muestra ayuda sobre el comando "x". También puede escribirse "git help x", para obtener ayuda con el comando "x". Si se ejecuta "git help" solo, imprime los comandos mas utilizados en Git y una corta descripción de cada uno.

  • git rm x

    Elimina en archivo "x" del espacio del trabajo. El archivo es eliminado tanto del espacio de trabajo como del disco duro, por lo que debe usarse con cuidado.

    El argumento --cached sirve para borrar elementos del staging sin eliminarlos del espacio de trabajo (como si se hubieran incluido por accidente).

    El argumento -n, ejecuta una "corrida en seco": muestra si puede borrar un archivo "x" o si están en el staging. Solo puede eliminar archivos previamente confirmados.

    El argumento -f, fuerza la eliminación de un archivo, sin importar donde esté o si tiene cambios.

    La opcion -r, permite eliminar recursivamente archivos, como las carpetas que tienen otros archivos dentro.

  • git rm vs. rm

    Mientras que "git rm" es una de las funciones de Git, el comando "rm" por si solo es una funcion de los entornos Linux/Unix para eliminar archivos. De esta forma, git rm manipula archivos del staging y el espacio de trabajo, mientras que rm suele usarse fuera del entorno de Git.

  • git commit

    Permite confirmar cambios en el repositorio local. Con el argumento: -m "x", puede agregarse un mensaje explicativo "x" a la confirmación (entre comillas). Es una mala práctica no hacerlo.

    Si se indica la bandera -a, los archivos con cambios fuera del staging se agregan de forma automática, sin tener que hacer git add.

    Si se intenta enviar un commit sin mensaje se abre el editor de texto Vim (o VS Code para algunos), solicitando el mensaje de la confirmación.

  • El Editor Vim

    Vim es el editor predeterminado de Git, que abre al escribir "vim" en Git Bash o cuando Git necesita que el usuario escriba un mensaje. Para escribir en Vim, primer se presiona la tecla "i" y luego se escribe el mensaje. Con el comando " Esc + Shift + ZZ" se guardan los cambios y se sale de Vim.
